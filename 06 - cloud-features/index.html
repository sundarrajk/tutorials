The Service Models

IaaS · PaaS · SaaS

Three layers of cloud service — how much you manage vs. how much the provider manages

IaaS — Infrastructure as a Service
  • Provider manages: physical servers, network, storage, hypervisor
  • You manage: OS, runtime, middleware, application, data
  • Most flexible — full control over the environment
  • Examples: AWS EC2, Azure Virtual Machines, GCP Compute Engine
PaaS — Platform as a Service
  • Provider manages: everything up to and including the runtime
  • You manage: your application code and data only
  • Deploy code — not servers. No OS patching needed.
  • Examples: AWS Elastic Beanstalk, Azure App Service, GCP App Engine
SaaS — Software as a Service
  • Provider manages everything — hardware, OS, runtime, application
  • You use the software via a browser — zero infrastructure ownership
  • Pay per user per month — the Salesforce model
  • Examples: Microsoft 365, Google Workspace, Salesforce, Zoom
The Key Shift: CAPEX (buy hardware upfront, depreciate over 5 years) → OPEX (pay monthly for what you use). No more idle servers. Scale up in minutes, scale down and stop paying. The CFO and the CTO both win.
The Market

The Three Cloud Giants

AWS, Azure, and GCP together hold ~67% of the global cloud market — with different strengths

☁ AWS — The Pioneer
  • Launched 2006 — first mover advantage, largest ecosystem
  • ~31% market share · 200+ services · 33 regions · 105 AZs
  • Broadest service catalogue; richest managed service options
  • Strongest for: startups, internet companies, broadest service depth
  • Key differentiators: Lambda (serverless), SageMaker (ML), Aurora (DB)
☁ Azure — The Enterprise Leader
  • Launched 2010 — deepest Microsoft/Office 365 integration
  • ~25% market share · 200+ services · 60+ regions
  • Dominant in enterprises already running Windows Server, Active Directory
  • Key differentiators: AVD (Virtual Desktops), Azure DevOps, Arc (hybrid)
  • Strongest for: enterprise hybrid cloud, Microsoft ecosystem
☁ GCP — The AI/Data Leader
  • Launched 2008 — runs on Google's own global fibre network
  • ~11% market share · 150+ services · 40 regions
  • Invented Kubernetes (GKE is the reference implementation)
  • Key differentiators: BigQuery, Vertex AI, GKE, global network
  • Strongest for: data analytics, AI/ML workloads, Kubernetes

Service Category 5

Messaging, Queues & Events

Decouple services, buffer load spikes, and build event-driven architectures without managing brokers

Why Messaging Matters
  • Decoupling: Service A sends a message; Service B processes it independently
  • Load levelling: absorb traffic spikes — queue fills, workers drain at their own rate
  • Reliability: if Service B is down, messages wait safely in the queue
  • Fan-out: one event triggers multiple downstream services in parallel
Message Queue Patterns
  • Queue (point-to-point): one producer → one consumer — order processing
  • Pub/Sub (broadcast): one event → many subscribers — event notifications
  • Streaming: high-throughput ordered log — real-time analytics, audit trails
  • Dead letter queue: failed messages held for inspection and retry

Service Category 8

Security & Identity

IAM, secrets management, encryption, threat detection, and compliance — the security stack built in

Identity & Access Management
  • AWS IAM / Azure Active Directory (Entra ID) / GCP IAM
  • Fine-grained permissions: who can do what on which resource
  • Service accounts / managed identities — apps authenticate without passwords
  • MFA, SAML federation, SSO with corporate identity providers
  • Azure Entra ID: deeply integrated with Microsoft 365 and on-prem AD
Secrets & Encryption
  • AWS Secrets Manager / Azure Key Vault / GCP Secret Manager
  • Store API keys, DB passwords, certificates — never in source code
  • Automatic rotation of credentials
  • KMS (Key Management Service): encrypt data at rest and in transit
  • Hardware Security Modules (HSM) for compliance (PCI-DSS, HIPAA)
Threat Detection
  • AWS GuardDuty / Azure Defender / GCP Security Command Center
  • Continuous monitoring for unusual API calls, network patterns, anomalies
  • Automated response: quarantine a compromised instance automatically
  • DDoS protection built-in at network edge
Compliance & Governance
  • All three meet: SOC 2, ISO 27001, PCI-DSS, HIPAA, GDPR
  • AWS Config / Azure Policy / GCP Org Policy: enforce rules across all resources
  • Audit logs: every API call logged — who did what and when
  • Private networking: VPC endpoints keep data off the public internet

Summary

Key Takeaways

What to remember about cloud services

① Service Model Determines Control
  • IaaS = maximum control, maximum responsibility. SaaS = zero control, zero responsibility. PaaS lives in between. Choose based on how much you want to manage.
② Three Giants, Different Strengths
  • AWS: broadest catalogue, best for startups and internet companies. Azure: best for Microsoft/enterprise shops. GCP: best for AI/ML, data analytics, Kubernetes.
③ Managed Services = No Ops Toil
  • Every managed service (RDS, ElastiCache, EKS…) eliminates patching, backups, failover setup, and capacity planning. Your team focuses on application logic, not infrastructure plumbing.
④ Serverless = Pay for Actual Use
  • Lambda/Functions/Cloud Run charge per invocation, often at millisecond granularity. A service handling 1,000 requests/month costs cents. No minimum, no idle cost.
The Cloud is a Superpower: A two-person startup today can access the same global infrastructure, AI models, and managed database technology as the largest enterprises in the world — billed by the second, with no upfront commitment.
